# 幹事王

**デモ: https://kanjioh.pages.dev/**

飲み会・合コン・旅行などの幹事向け精算額計算ツール。参加者ごとの「傾斜（支払い比率）」を設定し、各次会の総額から自動で精算額を計算します。

## 機能

- **傾斜計算**: 参加者ごとに支払い比率（傾斜）を設定。役職・立場に応じた不均等な負担割り当てが可能
- **複数次会対応**: 1次会・2次会・3次会…とまとめて計算
- **LRM丸め**: 指定した単位（1・100・500・1000円）に丸め、合計が総額と一致するよう最大剰余法で端数を調整
- **幹事優遇保証**: 幹事が同傾斜の参加者より少ない金額にならないよう警告と自動提案
- **URL共有**: 入力内容をURLハッシュに保存して共有。サーバー不要

## 使い方

### 1. 参加者を登録

「参加者追加」ボタンで行を追加し、以下を入力します。

| 項目 | 説明 |
|------|------|
| お名前 | 参加者の名前 |
| 傾斜 | 支払い比率の重み（例: 部長=5、一般=3、新人=2） |
| 幹事 | 端数を引き受ける人を1人だけラジオボタンで選択 |
| 出欠 | 各次会への出欠チェックボックス |
| ご寄付額 | 会費を前払いしている場合などに入力（精算額から差し引かれます） |

### 2. 総額を入力

画面上部の総額パネルに、各次会の実際の支払総額を入力します。「次会追加」ボタンで3次会以降も追加できます。

### 3. 丸め単位を選んで計算

ドロップダウンで丸め単位（1・100・500・1000円）を選び、「計算」ボタンを押します。

---

## 傾斜計算の仕組み

各次会 $k$ における参加者 $i$ の負担額（share）を以下のように計算します。

$$\text{share}_{i,k} = \text{総額}_k \times \frac{\text{傾斜}_i \times \text{出欠}_{i,k}}{\sum_j \text{傾斜}_j \times \text{出欠}_{j,k}}$$

ご寄付がある場合は `精算額 = max(0, share - 寄付額)` で差し引きます。

### 丸め処理（最大剰余法 LRM）

1. **基本丸め**: 各精算額を指定単位で `Math.round`
2. **誤差計算**: 丸め後の合計と総額の差（`diff`）を計算
3. **グループ調整**: 同傾斜の参加者をグループとして扱い、グループ単位で `±1単位` を調整（グループ内は同額を維持）
4. **端数吸収**: 1単位未満の残差は幹事が吸収

### 幹事の優遇保証

理想状態の定義：
- 幹事の精算額 ≥ 同傾斜の非幹事の精算額（幹事が損をしない）
- 幹事の精算額 ≤ 上位傾斜の参加者の精算額（傾斜の序列を維持）

条件を満たせない場合、警告モーダルを表示し、解消できる小さい丸め単位への変更を提案します（1円単位が最小）。

## URL共有の仕組み

「共有」ボタンを押すと、入力データを以下の形式でエンコードしてURLのハッシュ（`#`）に保存します。

```
https://example.com/#<Base64エンコードされたJSON>
```

`#` 以降はサーバーに送信されないため、データはブラウザ内で完結します。共有されたURLを開くと自動的に状態が復元されます。

### 保存されるデータ

```json
{
  "n": 2,
  "u": 1000,
  "ti": "20260604",
  "t": [30000, 15000],
  "p": [
    { "n": "田中", "s": "5.0", "k": true,  "a": [1, 1], "d": [0, 0] },
    { "n": "山田", "s": "3.0", "k": false, "a": [1, 0], "d": [500, 0] }
  ]
}
```

| キー | 内容 |
|------|------|
| `n` | 次会数 |
| `u` | 丸め単位 |
| `ti` | タイトル |
| `t` | 各次会の総額 |
| `p` | 参加者配列（n=名前, s=傾斜, k=幹事, a=出欠[], d=寄付[]） |
